AlocasiaTrack

Plant inventory, sales tracking & marketplace integrations

v1.0

AlocasiaTrack is a desktop app for collectors and small-scale sellers of Alocasia (and other tropical plants). It keeps track of your stock of plants, pups, and corms; records sales; syncs listings to Facebook Marketplace and Shopify; backs up data to Google Sheets and Google Drive; and includes a built-in web server so you can access your inventory from a phone or tablet.

Installation

Using the Installer (recommended)

  1. Run AlocasiaTrackSetup.exe.
  2. Follow the setup wizard. No administrator rights are required.
  3. Optionally tick Create desktop shortcut.
  4. Click Finish — the app will launch automatically.

macOS

  1. Open AlocasiaTrackSetup_v1.0.0.dmg.
  2. Drag AlocasiaTrack.app into your Applications folder.
  3. Launch from Applications or Spotlight.
On first launch macOS may show a security warning. Go to System Settings → Privacy & Security and click Open Anyway.

Running from Source

If you have Python 3.11+ installed:

pip install -r requirements.txt
python main.py

System Requirements

  • Windows 10 / 11 (64-bit) or macOS 12+ (Intel or Apple Silicon via Rosetta 2)
  • ~300 MB disk space (including app and bundled Python)
  • Internet connection required for Google, Facebook, and Shopify integrations

First Launch

On first launch AlocasiaTrack creates its database and configuration folder automatically at: 

Windows:  %LOCALAPPDATA%\AlocasiaTrack\
macOS:    ~/Library/Application Support/AlocasiaTrack/

This folder contains inventory.db (your plant data) and config.json (your settings). Nothing is stored inside the install directory.

Facebook Marketplace prompt

Shortly after launch you may see a dialog asking whether to install Playwright. Playwright is the browser automation tool that powers the Facebook Marketplace integration. If you do not plan to use Facebook Marketplace, click Skip for now — you will not be asked again. You can install it later from Settings → Facebook → Browser Requirement.

The app ships with a pre-loaded species library of 100+ Alocasia cultivars. You can start adding stock straight away without configuring any integrations.

Dashboard

The Dashboard is the home screen. It refreshes automatically every time you navigate to it.

Stat cards

CardMeaning

Monthly stats

The current month's revenue (total sale prices), profit (revenue minus cost basis and shipping), and number of sales are shown below the cards.

By-type breakdown

A small row of cards shows how your stock is split between Plants, Pups, and Corms.

Recent sales

The bottom of the Dashboard lists your ten most recent sales with date, SKU, species, sale price, and profit at a glance.

Stock Management

The Stock view is where you manage everything that is for sale. Each item is one of three types:

  • Plant — a rooted, potted plant in a growable stage
  • Pup — an offset or division taken from a mother plant
  • Corm — an unrooted bulb/rhizome cutting

Adding a stock item

  1. Click + Add Item in the top-right of the Stock view.
  2. Choose the item type: Plant, Pup, or Corm.
  3. Set the Quantity (default 1). Increase this to create several identical items at once — each will receive its own unique SKU.
  4. Select the Species using the searchable dropdown — start typing any part of the name to filter the list instantly.
  5. If this item came from one of your mother plants, search and select it under Parent Plant.
  6. Fill in Growth Stage, Condition, and Pot Size.
  7. Enter an Asking Price and optionally a Cost Basis (what you paid for it).
  8. Add any notes, then click Save.
Use the Quantity field when adding a batch of propagations — for example, set Quantity to 10 to create 10 corms at once. All items share the same fields but each gets its own SKU (CRM-0001 through CRM-0010). You can edit them individually afterward.
AlocasiaTrack assigns each item a unique SKU automatically (e.g. PLT-0001, PUP-0003, CRM-0002). SKUs are permanent and never reused.

Searchable species dropdown

Both the Species and Parent Plant fields use a searchable type-ahead dropdown rather than a plain list:

  • Click the field or the ▾ arrow to open the full list.
  • Start typing to filter — for example, typing dra narrows the list to all entries containing those letters.
  • Use the ↑ / ↓ arrow keys to move through the filtered list, then press Enter to select.
  • Press Escape or click elsewhere to close without selecting.

Adding photos

Inside the stock item dialog there is a Photos panel. Click + Add Photos to browse for one or more images. Photos are stored locally on your machine. If Google Drive is configured, they are also uploaded automatically.

  • Supported formats: JPG, PNG, WEBP, GIF, BMP
  • Click any thumbnail to view it full-size
  • Right-click (or use the ✕ button) to remove a photo

Each stock item moves through these statuses:

SoldSold to a buyerClick Record Sale in the toolbar

Filtering the stock list

Use the filter controls at the top of the Stock view to narrow the list by Type (Plant / Pup / Corm), Status (Available / Listed / Sold / etc.), and Species. Filters can be combined — for example, show only Available Pups of a specific cultivar.

Listing on Shopify

If Shopify is configured, select any stock item and click the green Shopify button in the toolbar. AlocasiaTrack creates or updates the product in your Shopify store and sets the item's status to Listed.

Listing on Facebook Marketplace

If the Facebook session is active, select any stock item and click the blue Post to FB button in the toolbar. A dialog lets you review and edit the pre-filled title, price, condition, and description. When you click Post to Marketplace, Chromium opens the listing form with everything filled in. Review the page, make any final adjustments, then click Publish.

Mother Plants

The Mother Plants view tracks your personal collection — the plants you keep and propagate from, rather than selling directly.

Adding a mother plant

  1. Click + Add Plant.
  2. Search and select the species using the searchable dropdown.
  3. Give the plant a nickname (e.g. "Big Frydek").
  4. Click the calendar icon next to the date field to pick an acquisition date from the calendar picker, or type the date manually in YYYY-MM-DD format.
  5. Set the pot size, health status, and location.
  6. Add photos if desired, then click Save.

Date picker

All date fields in the app include a 📅 calendar button. Click it to open a month-view calendar:

  • Use ◀ / ▶ to navigate between months.
  • Click any day to select it — the field fills with YYYY-MM-DD automatically.
  • Click Today at the bottom to jump straight to today's date.
  • You can also type a date directly in the text field if you prefer.

Linking stock to a mother plant

When adding a Pup or Corm to Stock, the Parent Plant searchable dropdown lists all your mother plants. Selecting one lets you trace every propagation back to its source plant.

Health statuses

  • Healthy — actively growing, no issues
  • Recovering — rebounding from stress or repotting
  • Sick — showing pests, rot, or significant decline
  • Dormant — resting, no active growth

Species & Cultivars

The Species & Cultivars view is a reference library. It ships pre-loaded with 100+ Alocasia species and cultivars, including their typical care levels and market price ranges.

Columns

ColumnMeaning
Display NameThe full trade name used in listings — always includes "Alocasia", e.g. "Alocasia Dragon Scale"
Common NameThe popular nickname only, without the genus — e.g. "Dragon Scale", "Polly", "Giant Taro"
Scientific / Cultivar NameThe botanical epithet — e.g. baginda 'Dragon Scale'
Care LevelEasy, Moderate, or Difficult
Price Min / MaxTypical retail price range for corms through mature plants
In StockHow many Available or Listed items you currently have of this species
NotesShort care description (first 60 characters shown; full notes visible when editing)

Naming conventions

Three separate name fields keep things unambiguous:

  • Scientific name — the botanical epithet without genus, e.g. baginda 'Dragon Scale'
  • Common name — the short nickname collectors use, without "Alocasia", e.g. "Dragon Scale"
  • Display name — the trade / listing name with "Alocasia", e.g. "Alocasia Dragon Scale"
The display name is what appears in stock dropdowns and Facebook listing titles. The common name is the pure nickname and never includes the genus.

Searching the species list

A search bar at the top of the Species view filters all columns simultaneously as you type. Searching for "dragon" instantly narrows the list to every entry where any column contains that word. A ✕ button appears while the search is active; click it to clear.

Sorting columns

Left-click any column heading to sort by that column. Click again to reverse the order. An ▲ or ▼ indicator in the heading shows the active sort direction.

Filtering columns

Right-click any column heading to open a filter popup for that column. The popup shows all unique values in that column as checkboxes:

  • Uncheck any values you want to hide, then click Apply.
  • A ⊘ indicator in the column heading shows when a filter is active.
  • Click Clear Filter to remove that column's filter.
  • Multiple column filters stack — for example, show only "Easy" care level items that have zero in stock.

The status bar below the table updates to show Showing X of Y cultivars whenever any filter is active.

Importing species lists

Click ⬇ Import List (blue button, top-left of the toolbar) to open the Import Species Lists dialog. This lets you populate your database from pre-built species packs without typing anything manually.

  1. The dialog shows all available species packs — the built-in Alocasia pack ships with the app.
  2. Each pack card previews exactly how many entries are New, Updates, or No change before you commit.
  3. Tick the packs you want and click Import Selected.
  4. The status bar shows a live count as entries are processed. When complete, the species list refreshes automatically.
You can add more genus packs (e.g. Philodendron, Monstera) by dropping a .py file into the species_seeds/ folder inside the application directory. The new pack appears in the Import dialog automatically — no code changes needed. See the species_seeds/alocasia.py file for the expected format.

Adding a custom cultivar

  1. Click + Add Species.
  2. Enter the Scientific / Cultivar Name (required) — e.g. amazonica 'Polly'.
  3. Enter the Display Name — the trade name with genus, e.g. Alocasia Polly.
  4. Enter the Common Name — the short nickname without genus, e.g. Polly.
  5. Set the care level and typical price range.
  6. Add notes if desired, then click Save.
Price ranges here are for reference only. The actual asking price is set on each individual stock item and can differ from the cultivar's range.

Recording Sales

From the Stock view (recommended)

  1. Find the item in the Stock list.
  2. Click the Record Sale button in the toolbar (the item must be selected).
  3. Enter the sale price, buyer name, buyer contact, shipping method, and shipping cost.
  4. Use the calendar icon next to the sale date field to pick the date, or type it directly.
  5. The platform field is pre-filled if the item had an active listing.
  6. Click Record Sale.

AlocasiaTrack automatically calculates profit as:
Profit = Sale Price − Cost Basis − Shipping Cost

After saving the sale, the stock item's status is set to Sold, any active Shopify listing is unpublished, and a Gmail notification is sent (if configured).

The Sales view

The Sales tab shows every recorded sale with totals for revenue, profit, and count, broken down by the current month and all time.

Sales records are permanent. If you make a mistake, edit the sale record directly from the Sales view.

Google Integration Setup

The Google Sheets, Drive, and Gmail integrations all share the same Google OAuth credentials. You only need to set this up once.

Step 1 — Create a Google Cloud project

  1. Go to console.cloud.google.com and sign in.
  2. Click Select a project → New Project. Name it anything (e.g. "AlocasiaTrack").
  3. In the left menu go to APIs & Services → Library.
  4. Enable these three APIs: Google Sheets API, Google Drive API, Gmail API.

Step 2 — Create OAuth credentials

  1. Go to APIs & Services → Credentials.
  2. Click + Create Credentials → OAuth client ID.
  3. If prompted, configure the OAuth consent screen first: choose External, fill in the app name, and add your email as a test user.
  4. Back on Create Credentials: choose Application type = Desktop app, click Create.
  5. Click Download JSON and save the file somewhere easy to find (e.g. your Desktop).

Step 3 — Connect in AlocasiaTrack

  1. Open AlocasiaTrack and go to Settings → Google Sheets.
  2. Click Browse… next to Credentials file and select the JSON you downloaded.
  3. Click Connect Google Account.
  4. A browser window will open — log in to your Google account and click Allow.
  5. The browser closes and the app shows Connected.
The OAuth token is saved locally so you only need to authorise once. If you ever see an "Authorisation failed" error, click Connect again to refresh.
Keep your credentials.json file private. It is your app's identity key for Google services.

Google Sheets Integration

AlocasiaTrack can mirror your inventory to a Google Spreadsheet. This gives you a shareable, always-up-to-date record that you can view on any device.

Creating the spreadsheet

  1. Go to Settings → Google Sheets.
  2. Click Create New Sheet — AlocasiaTrack creates a spreadsheet in your Google Drive named "AlocasiaTrack Inventory".
  3. The Spreadsheet ID is saved automatically.

Alternatively, paste the ID of an existing spreadsheet into the Spreadsheet ID field and click Save. (The ID is the long string in a Google Sheets URL between /d/ and /edit.)

Syncing data

  • Click Sync Now to push the current inventory to the sheet.
  • The sheet is updated in full each time — existing data is replaced.
  • Three tabs are written: Stock, Plants, and Sales.
Sync is manual — the sheet does not update in real time. Run Sync after making significant changes.

Google Drive Photo Backup

When Google Drive is configured, photos you attach to stock items are automatically copied to your Google Drive in an organised folder structure.

Setup

  1. Complete the Google Integration Setup above.
  2. Go to Settings → Google Drive.
  3. Optionally, enter a Root Folder ID if you want photos stored inside a specific Drive folder. Leave blank to use the root of your Drive.
  4. Click Save.

Gmail Notifications

AlocasiaTrack can send you email notifications for two events: a sale being recorded and your stock of a particular species running low.

Setup

  1. Complete the Google Integration Setup above.
  2. Go to Settings → Gmail Alerts.
  3. If the Gmail scope is not yet authorised, click Re-authorize — a browser window opens once for you to grant the Gmail permission.
  4. Enter the email address where you want to receive alerts.
  5. Toggle Sale alerts and/or Low-stock alerts on.
  6. For low-stock alerts, set the threshold (default: 2 — notified when the available count drops to or below this number).
  7. Click Save.

Sale alert email

Sent immediately when you record a sale. Includes:

  • Item SKU, species, and type
  • Sale price, shipping cost, and calculated profit
  • Buyer name and contact
  • Platform and sale date

Low-stock alert email

Sent after recording a sale when the remaining Available count for that species falls to or below your threshold. Useful if you want to propagate more before running out.

Facebook Marketplace Integration

AlocasiaTrack uses Playwright (a browser automation library) to open a real Chromium browser, navigate to your Marketplace listing form, and fill in the details automatically. You remain in control — the browser is visible and you can review or change anything before submitting.

Installing Playwright

Playwright is an optional component and is not bundled with the installer (it requires a ~200 MB Chromium browser download). Install it through the app:

  1. Go to Settings → Facebook.
  2. Scroll to the Browser Requirement section.
  3. Click Install Playwright + Chromium and wait for the download to complete (~2 minutes on a fast connection).
  4. The status updates to Playwright + Chromium installed when complete, and the Login button becomes active.

To install manually, open a terminal and run:

pip install playwright
playwright install chromium
The Chromium download is approximately 150–200 MB. Make sure you are on Wi-Fi before installing.
When running from the installed .exe, Playwright runs using your system Python installation (not the bundled Python). This is intentional — Playwright and Chromium are too large to bundle. The Login to Facebook button is automatically disabled when Playwright is not detected, preventing confusing error messages.

Connecting your Facebook account

  1. Go to Settings → Facebook.
  2. Click Login to Facebook.
  3. A Chromium browser window opens and navigates to facebook.com.
  4. Log in normally (including any 2-factor authentication).
  5. Once you are logged in, the app detects the session and saves it automatically.
  6. The Settings page shows Session saved — you are now connected to Facebook.
The session is saved as cookies in %LOCALAPPDATA%\AlocasiaTrack\fb_session.json. It typically stays valid for several weeks. If Marketplace posts start failing, click Clear Session and reconnect.

Posting a listing

  1. Find and select the item in the Stock view.
  2. Click the blue Post to FB button in the toolbar.
  3. A dialog appears pre-filled with the title, price, condition, and description from your stock data. Edit any field if needed, then click Post to Marketplace.
  4. Chromium opens and navigates to the Marketplace create listing form with everything filled in.
  5. Any photos attached to the stock item are uploaded automatically.
  6. Review the form, make any final adjustments, then click Publish.
  7. The browser closes and the listing URL is saved to your records.

Default hashtags & location

Go to Settings → Facebook → Listing Defaults to set a default location (city or zip) and hashtags that are appended to every listing description automatically.

Disconnecting

Click Clear Session in Settings → Facebook to delete the saved cookies. You will need to reconnect before posting again.

Shopify Integration

AlocasiaTrack connects to your Shopify store using the Admin REST API. It can create or update products for individual stock items, or sync all Available items at once.

Getting your Shopify API credentials

  1. Log in to your Shopify admin panel.
  2. Go to Settings → Apps and sales channels → Develop apps.
  3. Click Create an app. Give it a name like "AlocasiaTrack".
  4. Under Configuration, click Configure Admin API scopes.
  5. Enable these scopes: read_products, write_products.
  6. Click Save, then go to API credentials.
  7. Click Install app, then copy the Admin API access token (shown only once — save it somewhere safe).

Connecting in AlocasiaTrack

  1. Go to Settings → Shopify.
  2. Enter your Shop URL (e.g. your-store.myshopify.com).
  3. Paste your Access Token into the token field.
  4. Click Save for each field.
  5. Click Test Connection to verify the credentials work.

Syncing a single item

Click the green Shopify button on any Available stock row. The product is created in your Shopify store (or updated if it was previously synced). The item's status changes to Listed.

Syncing all Available items

Go to Settings → Shopify and click Sync All Available. This creates or updates a Shopify product for every item currently in Available status.

Auto-unpublish on sale

When you record a sale for an item that was synced to Shopify, AlocasiaTrack automatically sets the Shopify product to Draft (unpublished) so it disappears from your storefront.

AlocasiaTrack stores a local mapping of stock IDs to Shopify product IDs in %LOCALAPPDATA%\AlocasiaTrack\shopify_products.json. Do not delete this file — it prevents duplicate products from being created.

General Settings

Go to Settings → General to customise app-wide options that affect how the stock dialogs work.

Pot Sizes

The pot size list controls which options appear in the Pot Size dropdown in the Add / Edit Stock and Add / Edit Mother Plant dialogs. The default list is: 

No pot  ·  2"  ·  4"  ·  6"  ·  8"  ·  10"  ·  12"+

Adding a pot size

  1. Type the new size in the text entry at the bottom of the pot size list (e.g. 3" or Growbag).
  2. Click + Add. The new option appears at the bottom of the list.

Reordering pot sizes

Use the ↑ and ↓ arrow buttons on any row to move it up or down. The order here is the order that appears in the dropdowns.

Removing a pot size

Click the ✕ button on any row to remove it. Stock items that already used that size are not affected — only new items will no longer see it as an option.

Resetting to defaults

Click Reset to Defaults to restore the original seven pot size options. Any custom sizes you added will be removed.

Changes to the pot size list take effect immediately for any dialog you open after saving.

Web App & Mobile Access

AlocasiaTrack includes a built-in web server that makes your inventory accessible from any browser on your local network — your phone, tablet, or another PC. The web app reads and writes the same database as the desktop app in real time.

Starting the web server

  1. Go to Settings → Web App.
  2. Click Start Web Server.
  3. The status changes to ● Running and a URL appears, e.g. http://192.168.1.45:5000.
  4. Open that URL in any browser on the same Wi-Fi network.
Click the URL label directly in Settings to open the web app in your default browser. Click Copy URL to copy it to the clipboard for sharing or bookmarking on your phone.

Accessing from your phone

With the server running, open the address shown in Settings in your phone's browser. For the best experience on mobile:

  • Android: In Chrome, tap the three-dot menu → Add to Home screen. The app installs as a PWA and opens without browser chrome.
  • iPhone / iPad: In Safari, tap Share → Add to Home Screen. Same result — it behaves like a native app.

Web app features

ScreenWhat you can do
DashboardView all inventory stats, monthly financials, and recent sales
StockSearch and filter your inventory; view item details including photos; record sales via a modal form
Stock DetailSee all item fields; tap any photo thumbnail to view full-size; change status with one tap
SalesBrowse all sales history with monthly and all-time totals
SpeciesBrowse and search the species reference library
Mother PlantsBrowse your personal plant collection

Port configuration

The default port is 5000. To change it:

  1. Enter a new port number (1024–65535) in the Port field.
  2. Click Save & Restart — the server restarts automatically on the new port.
Change the port if port 5000 is already in use by another application on your PC, or if you prefer a specific port (e.g. 8080).

Auto-start

Tick Start web server automatically when the app launches to have the server start silently every time you open AlocasiaTrack. The server runs in the background and does not affect desktop app performance.

Using a custom address

By default the server is accessible at your PC's local IP address (e.g. 192.168.1.45:5000), which changes if your router assigns a different IP. For a permanent memorable address like alocasia.local:

  1. Log in to your router's admin page.
  2. Under Static DHCP or Reserved IP, assign a fixed IP address to your PC's MAC address.
  3. Under Local DNS or Hostnames, add:
    alocasia.local → <your PC's fixed IP>

All devices on your Wi-Fi can then use http://alocasia.local:5000 as a permanent bookmark.

The web server is designed for trusted local networks only. Do not forward the port to the internet — there is no authentication.

Running the web app without the desktop

If you are running from source and want the web app only (e.g. on a headless server), you can launch it directly: 

python web_app/run.py

Troubleshooting

The app opens but shows no data

The app may be reading a different database than expected. Check that %LOCALAPPDATA%\AlocasiaTrack\inventory.db exists. If it does and the app is still empty, close the app, delete any __pycache__ folders inside the install directory, then relaunch.

Species list is empty or outdated

Click ⬇ Import List in the Species view, select the Alocasia pack, and click Import Selected. This will add any missing entries without removing your existing data. Alternatively, run the seed script from source: 

python scripts\seed_species.py

Google connection fails — "File not found" or "Invalid credentials"

  • Make sure the credentials file path set in Settings still exists.
  • Delete %LOCALAPPDATA%\AlocasiaTrack\token.json and reconnect — this forces a fresh OAuth flow.
  • Confirm the required APIs (Sheets, Drive, Gmail) are enabled in your Google Cloud project.

Facebook — "Login to Facebook" button is greyed out

The button is disabled until Playwright and Chromium are installed. Go to Settings → Facebook → Browser Requirement and click Install Playwright + Chromium. After installation the button becomes active automatically.

Facebook posting fails or the browser does not open

  • Confirm the Browser Requirement section in Settings → Facebook shows Playwright + Chromium installed.
  • If it still shows "not installed" after you installed it, try running playwright install chromium in a terminal to re-download the browser, then restart the app.
  • When running from the installed .exe, Playwright must be installed in your system Python (not the bundled one). Open a normal terminal (not inside the app) and run pip install playwright && playwright install chromium.
  • If you get a login wall every time, click Clear Session and reconnect.

Shopify "Test Connection" fails

  • Verify your shop URL is in the format your-store.myshopify.com (no https://).
  • Confirm the access token was copied completely — it starts with shpat_.
  • Check that the app in your Shopify admin has write_products scope enabled and is installed.

Photos are not uploading to Drive

  • Confirm Google Drive is enabled under Settings → Google Drive.
  • Make sure the Google account is connected (see Google Setup).
  • Drive uploads happen in the background — check back after a few seconds.

Web app not accessible from phone

  • Make sure both your PC and phone are on the same Wi-Fi network.
  • Check the server is running: Settings → Web App should show ● Running.
  • Use the IP address shown in Settings, not localhost — localhost only works on the PC itself.
  • If your Windows Firewall blocks the connection, allow Python through the firewall for private networks, or change the port and try again.

SKU numbers jumped unexpectedly

SKUs are assigned based on the total count of each item type ever added (including deleted items). This prevents gaps from causing duplicate SKUs if records are removed.

For any issue not listed here, run the app from source in a terminal (python main.py) to see Python error output directly.
